Introduction

I’m a front-end web developer/designer. I care about interfaces that feel obvious, copy that reduces friction, and code a teammate can read in one pass. This program shifted me from “it runs” to “it reads well, behaves predictably, and respects the user.” I can work across the stack when needed, but my best work lives on the surface—structure, semantics, accessibility, and the small interaction details that build trust.

Coursework gave me the habits I use every day: separate concerns, name things clearly, keep functions small, and write down why I made a choice. Mid-program classes sharpened my approach to data structures and algorithms and to disciplined input/edge-case handling, so behavior is predictable instead of accidental. Later work tied those habits to the web: present results coherently, keep interfaces forgiving, and document decisions so teammates (and future me) can move quickly. Outside of class, running an online store pushed me to prefer simple UX, good defaults, and outputs you can act on.
